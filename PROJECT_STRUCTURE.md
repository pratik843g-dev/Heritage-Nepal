## ğŸ“‚ Key Directories

### `/src/components/`
Reusable UI components used across multiple pages.

### `/src/pages/`
Individual page components that represent different routes in the app.

### `/src/data/`
Static data files containing heritage site information.

### `/secret/`
Standalone HTML tools for generating and testing QR codes offline. These don't require the main app to run.

### `/dist/`
Production build output (git-ignored). Generated by running `npm run build`.

## ğŸ”‘ Key Files

### `src/data/heritageSites.js`
The main database of heritage sites. Each site includes:
- Basic info (name, location, type, built year)
- Detailed descriptions
- Images and coordinates
- Visitor information
- Interesting facts

### `src/pages/ScannerPage.jsx`
Main scanner implementation with:
- Camera-based QR scanning
- File upload for QR images
- JSON and URL QR code parsing
- Error handling and debugging

### `src/App.jsx`
App routing configuration using React Router.

### Configuration Files
- `vite.config.js` - Vite bundler settings
- `tailwind.config.js` - Custom color theme (nepal-red, nepal-blue)
- `postcss.config.js` - PostCSS with TailwindCSS
- `netlify.toml` - Netlify deployment settings

## ğŸš€ Build Process

1. **Development**: `npm run dev`
   - Starts Vite dev server
   - Hot module replacement enabled
   - Runs on http://localhost:5173

2. **Production**: `npm run build`
   - Builds optimized bundle
   - Outputs to `/dist/`
   - Minifies and tree-shakes code

3. **Preview**: `npm run preview`
   - Preview production build locally
   - Tests the built app before deployment
